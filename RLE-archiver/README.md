# RLE Archiver

Простой архиватор на языке C, реализующий алгоритм сжатия RLE (Run-Length Encoding).  
Добавляет к сжатому файлу сигнатуру `RLE` для проверки формата при распаковке.

## Сборка

Требуется компилятор GCC и стандартная библиотека C.

```bash
gcc -o rle main.c rle.c -std=c99 -Wall -Wextra
```

## Использование
```bash
./rle -[e|d] <входной_файл>
```
`-e` – сжатие (упаковка)

`-d` – распаковка

Выходной файл создаётся автоматически путём добавления или удаления расширения .rle:
* при сжатии `file.txt` → `file.txt.rle`
* при распаковке `file.txt.rle` → `file.txt`

## Примеры

`./rle -e document.txt` - сжать файл

`./rle -d document.txt.rle` - восстановить сжатый файл

## Формат сжатого файла

Сжатый файл состоит из двух частей:
Заголовок – 3 байта `RLE`
Данные – последовательность пар `[счётчик][байт]`, где:

**счётчик** – 1 байт (1–255), указывающий количество повторений;

**байт** – 1 байт, значение повторяющегося символа.

Длинные серии разбиваются на несколько блоков.
Например, 300 одинаковых байт со значением `X` будут записаны как `[255][X] [45][X]`.

## Особенности и ограничения

* Программа **всегда пытается сжать данные** и записывает результат, даже если сжатый файл оказывается **больше исходного**.  
  Это штатное поведение для алгоритма RLE, который на неповторяющихся данных может увеличивать размер вплоть до удвоения.
* **Эффективность сжатия** зависит от типа данных:
  - Отлично сжимаются файлы с длинными сериями одинаковых байт (чёрно-белые BMP, простые текстуры, тексты с монотонными повторениями).
  - На уже сжатых данных (JPEG, PNG, ZIP) или случайном содержимом размер, скорее всего, увеличится.
* Исходный файл **полностью загружается в память** перед обработкой, поэтому для экстремально больших файлов (десятки гигабайт) может не хватить оперативной памяти.
* Максимальная длина непрерывной серии – 255 байт; более длинные серии автоматически разбиваются на несколько блоков..
